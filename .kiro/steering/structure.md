# Project Structure

## Directory Organization

```
mosaic/
├── convex/                    # Convex backend
│   ├── _generated/           # Auto-generated Convex types
│   ├── auth.config.ts        # Clerk authentication config
│   ├── schema.ts             # Database schema definitions
│   └── *.ts                  # Convex functions (queries, mutations)
├── src/
│   ├── app/                  # Next.js App Router
│   │   ├── layout.tsx        # Root layout with providers
│   │   ├── page.tsx          # Home page
│   │   └── globals.css       # Global styles
│   ├── components/
│   │   ├── ui/               # Reusable UI components (shadcn/ui)
│   │   ├── providers.tsx     # App-level providers (Clerk, Convex, Theme)
│   │   └── theme-provider.tsx
│   ├── features/             # Feature-based modules
│   │   └── auth/
│   │       └── components/   # Auth-specific components
│   ├── hooks/                # Custom React hooks
│   └── lib/
│       └── utils.ts          # Utility functions (cn, etc.)
└── public/                   # Static assets
```

## Architecture Patterns

### Feature-Based Organization
- Group related components, hooks, and logic by feature in `src/features/`
- Each feature can have its own `components/`, `hooks/`, `utils/` subdirectories

### Component Conventions
- UI primitives live in `src/components/ui/`
- Feature-specific components live in `src/features/{feature}/components/`
- Use "use client" directive for client components
- Server components by default (App Router)

### Convex Backend
- Schema definitions in `convex/schema.ts`
- Queries and mutations in separate files (e.g., `convex/projects.ts`)
- Never edit `convex/_generated/` - auto-generated by Convex

### Provider Pattern
- All providers wrapped in `src/components/providers.tsx`
- Order: ClerkProvider → ConvexProviderWithClerk → ThemeProvider → App content
- Authentication states handled via Convex components: `<Authenticated>`, `<Unauthenticated>`, `<AuthLoading>`

### Styling Conventions
- Use Tailwind utility classes
- Use `cn()` utility from `@/lib/utils` for conditional classes
- Component variants via `class-variance-authority`
- Theme variables defined in `globals.css`

## Import Conventions
- Use `@/` alias for imports from `src/`
- Example: `import { Button } from "@/components/ui/button"`
- Convex imports: `import { api } from "convex/_generated/api"`

## File Naming
- Components: PascalCase (e.g., `Button.tsx`, `UnauthenticatedView.tsx`)
- Utilities/hooks: kebab-case (e.g., `use-mobile.ts`, `utils.ts`)
- Convex files: kebab-case (e.g., `auth.config.ts`, `projects.ts`)
